# 思考类文章编写规则

## 文章定位

思考类文章旨在分享对某个话题的深度思考和独立见解，而非简单的知识传授。重点在于**观点的推导过程**、**多角度的分析**，以及**引发读者的思考**。

## 核心原则

### 1. 叙事性优先于列表式
- ✅ 使用流畅的段落叙述，让观点自然展开
- ✅ 通过故事、案例、类比来说明观点
- ❌ 避免过度使用分点列举（如：观点1、观点2、观点3）
- ❌ 避免大量使用表格、代码块来模拟图表

**示例对比：**

❌ **列表式（避免）：**
```
职业分化的三种路径：
1. AI 工具开发者
   - 技能要求：xxx
   - 薪资水平：xxx
2. 领域专家工程师
   - 技能要求：xxx
   - 薪资水平：xxx
```

✅ **叙事式（推荐）：**
```
我认为职业分化不是简单的二元对立。最顶端的 5% 会成为 AI 工具的开发者，
他们需要深厚的技术功底。但被很多人忽略的，是中间这 20% 的领域专家工程师。
他们的核心竞争力不是技术深度，而是技术和业务的结合...
```

### 2. 真实性和温度感
- ✅ 使用第一人称"我"来表达观点，增强真实感
- ✅ 承认不确定性（"我认为"、"可能"、"很难预测"）
- ✅ 分享思考的过程，而非只抛出结论
- ❌ 避免过于绝对的断言
- ❌ 避免像教科书或产品手册的冷冰冰语气

**示例：**
```
❌ "AI 将导致程序员岗位减少 30%"（过于绝对）
✅ "我的判断是，岗位总量在中长期可能持平或略降，取决于新需求的增长速度"
```

### 3. 结构清晰但不刻板
- ✅ 每个章节以问题或场景开头，自然引入讨论
- ✅ 保持逻辑推进的连贯性
- ✅ 适当使用小标题划分段落，但不要过度分层（最多二级标题）
- ❌ 避免过多的层级嵌套（1.1.1、1.1.2 这种）
- ❌ 避免每个段落都只有 2-3 句话

**推荐章节结构：**
```
## 引入性标题（带有悬念或问题）
开篇段落引入话题，说明为什么要思考这个问题...

接下来用 2-4 个段落展开论述，每个段落 4-8 句话...

中间可以穿插具体案例或类比...

最后用一个段落总结或过渡到下一章节...
```

### 4. 案例和类比的使用
- ✅ 用具体的场景或故事来说明抽象观点
- ✅ 使用历史类比帮助读者理解趋势（如：汽车 vs 马车、Excel vs 会计）
- ✅ 适当使用"想象一个..."来构建场景
- ❌ 避免堆砌概念和术语
- ❌ 避免只有论点没有论据

**示例：**
```
想象一个小型制造企业，他们可能需要一个生产管理系统来追踪订单、物料和进度。
在传统模式下，开发这样一个系统可能需要十几人的团队投入几个月时间，
成本可能高达数百万。对于一个年营收几千万的企业来说，这个投入显然太大了...
```

### 5. 可视化元素的克制使用
- ✅ 只在**确实需要对比**或**展示层次关系**时才使用列表
- ✅ 加粗用于强调关键观点，但不要每段都有
- ✅ 引用块用于突出核心洞察或重要结论
- ❌ 避免用代码块模拟图表（如金字塔、流程图）
- ❌ 避免大量使用 emoji 和特殊符号（偶尔用 ✅❌ 做对比除外）

**合理使用列表的场景：**
- 罗列客观事实（如：历史上发生的三次岗位整合）
- 总结性归纳（如：文章最后的核心判断）
- 对比两种方案的优缺点

**不应使用列表的场景：**
- 展开分析某个观点（应该用段落叙述）
- 描述某个概念（应该用自然语言解释）

### 6. 段落长度和节奏
- ✅ 段落长度适中，一般 4-8 句话
- ✅ 避免连续多个短段落（1-2 句话），会显得碎片化
- ✅ 偶尔使用单句段落来强调或转折
- ✅ 注意段落之间的过渡和呼应

**节奏示例：**
```
长段落（6-8 句）：展开分析某个核心观点
中等段落（4-5 句）：继续论述或提供补充论据
单句段落：这是一个需要严肃思考的问题。
中等段落（4-5 句）：引入下一个话题...
```

## 文章结构模板

### 开头（引入话题）
- 从一个问题、现象或个人经历开始
- 说明为什么这个话题值得思考
- 概述文章的思考路径（但不要列大纲）

### 正文（逐层展开）
- 每个大章节聚焦一个核心议题
- 用小标题概括章节主题，但内容用段落叙述
- 适当使用案例、数据、类比来支撑观点
- 承认不同视角，展示思考的复杂性

### 结尾（总结和行动）
- 回应开头提出的问题
- 总结核心判断（这时可以用简洁的列表）
- 提供可行的建议或启发
- 结束语要有力量感，但不要口号化

## Frontmatter 格式

```yaml
---
title: "文章标题：主题 + 视角/问题"
createDate: "YYYY-MM-DD"
updateDate: "YYYY-MM-DD"
tags:
  - 核心话题标签
  - 相关领域标签
  - 不超过 5 个
author: "作者名"
excerpt: "一句话概括文章的核心观点或问题，吸引读者阅读。120 字以内。"
---
```

## 语言风格

- **口语化但不失专业性**：用"我们"、"你"，但避免过度口语化
- **具体胜于抽象**：多用具体场景，少用大词
- **诚实的不确定性**：不要假装全知，坦诚表达"可能"、"大概率"
- **有温度的表达**：让读者感觉是在和一个真实的人对话

## 检查清单

在完成文章后，用以下问题自查：

- [ ] 是否过度使用了列表式内容？（超过 3 处大段列表就需要警惕）
- [ ] 段落是否流畅连贯，还是像知识点总结？
- [ ] 是否有具体的案例或故事支撑观点？
- [ ] 是否用了代码块来模拟图表？（应该删除，改用叙述）
- [ ] 语气是否像在和朋友分享思考，而非讲课？
- [ ] 核心观点是否清晰，读者能记住什么？
- [ ] 文章是否引发了思考，而非只是传递信息？
